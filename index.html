Walkthrough

Key Configurations

01
Fine-Grained Password Policy
Unlike the Default Domain Policy, Fine-Grained Password Policies allow different password rules for different users or groups.
PowerShell
New-ADFineGrainedPasswordPolicy `
  -Name "VMUsers-Password-Policy" `
  -Precedence 10 `
  -MinPasswordLength 10 `
  -ComplexityEnabled $true `
  -LockoutThreshold 5 `
  -LockoutDuration "00:30:00" `
  -MaxPasswordAge "90.00:00:00"

Add-ADFineGrainedPasswordPolicySubject `
  -Identity "VMUsers-Password-Policy" `
  -Subjects "nsimon"
✓ Verified
AppliesTo: CN=Neil Marvin Simon,OU=VMUsers,DC=mylab,DC=local
ComplexityEnabled: True | LockoutThreshold: 5 | MinPasswordLength: 10
02
Login Banner GPO
Applied domain-wide so every user sees the warning before logging in — verified working on the Windows 10 client.
PowerShell
Set-GPRegistryValue -Name "Security Policy - Login Banner" `
  -Key "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" `
  -ValueName "legalnoticecaption" `
  -Type String -Value "AUTHORIZED ACCESS ONLY"

Set-GPRegistryValue -Name "Security Policy - Login Banner" `
  -Key "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" `
  -ValueName "legalnoticetext" `
  -Type String -Value "This system is for authorized users only..."
✓ Result on Win10 Client
Login banner displayed: "AUTHORIZED ACCESS ONLY"
Message visible before credentials entered ✅
03
Network Drive Mapping (CompanyShare)
Created a shared folder on WinServer2022 and mapped it as Z: drive on the Windows 10 client for domain users.
PowerShell — Create Share
# Create folder structure
New-Item -Path "C:\CompanyShare\IT" -ItemType Directory
New-Item -Path "C:\CompanyShare\Users" -ItemType Directory
New-Item -Path "C:\CompanyShare\General" -ItemType Directory

# Share with permissions
New-SmbShare -Name "CompanyShare" -Path "C:\CompanyShare"
Grant-SmbShareAccess -Name "CompanyShare" `
  -AccountName "Domain Admins" -AccessRight Full -Force
Grant-SmbShareAccess -Name "CompanyShare" `
  -AccountName "Domain Users" -AccessRight Read -Force

# Manual mapping on client
net use Z: \\192.168.1.200\CompanyShare /persistent:yes
✓ Result
Z: drive mapped to \\192.168.1.200\CompanyShare
Folders visible: IT, Users, General ✅
04
Temporary Privilege Elevation
When IT needs to fix something on a user's computer without logging them out — using "Run as different user" technique.
CMD — Run as Admin while user is logged in
# Open Control Panel as Administrator while nsimon is logged in
runas /user:mylab\Administrator "control"

# Or temporarily disable a GPO from the server
Set-GPLink -Name "Desktop Policy - Restrictions" `
  -Target "OU=VMUsers,DC=mylab,DC=local" `
  -LinkEnabled No
Troubleshooting

Issues & Fixes

DisableCMD persisted after removal
Cause
CMD was blocked by two GPOs simultaneously — removing it from one left it active from the other.
✓ Fix
Used Get-GPRegistryValue to identify which GPO had the active setting, then removed it from the correct policy.
Invoke-GPUpdate Remote Timeout
Cause
Windows 10 firewall blocking Remote Scheduled Tasks Management port.
✓ Fix
Ran gpupdate /force directly on the Win10 client instead of using remote update from the server.
Administrator Login Failure After OU Move
Cause
Password cache conflict after moving Administrator account to VMAdmin OU.
✓ Fix
Reset password using net user Administrator and re-enabled account with /active:yes.
Z: Drive Not Auto-Mapping via GPO
Cause
Registry-based drive mapping method requires specific GPO preferences configuration.
✓ Fix
Used net use command as workaround with /persistent:yes to maintain mapping across reboots.
